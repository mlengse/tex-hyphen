# Contributing to tex-hyphen

Contributions are welcome. Please discuss larger changes on the
[tex-hyphen mailing list](http://tug.org/mailman/listinfo/tex-hyphen) before
submitting a pull request.

---

## Adding a new language

### 1. Create the pattern file

Create `hyph-<lang>.tex` in `hyph-utf8/tex/generic/hyph-utf8/patterns/tex/`.
The file must begin with a YAML metadata header (lines starting with `%`) followed
by the `\patterns{ ... }` block.

Minimum required header fields:

```yaml
% title: Hyphenation patterns for <Language>
% copyright: Copyright (C) <year> <author>
% notice: This file is part of the hyph-utf8 package.
%     See http://www.hyphenation.org/tex for more information.
% language:
%     name: <Language name>
%     tag: <BCP 47 tag>
% authors:
%   -
%     name: <Author name>
% licence:
%     text: <Licence text>
% hyphenmins:
%     typesetting:
%         left: <minimum chars before hyphen>
%         right: <minimum chars after hyphen>
```

Optional fields:

- `version:` — date string (e.g. `2024-01-15`)
- `changes:` — changelog entries
- `texlive:` — TeX Live integration metadata:
  - `encoding:` — character encoding (e.g. `utf8`, `ascii`, `iso8859-1`)
  - `babelname:` — Babel language name (if applicable)
  - `legacy_patterns:` — legacy pattern filename (if applicable)
  - `message:` — human-readable description
  - `package:` — TeX Live package name
- `known_bugs:` — known issues

### 2. Validate the header

```sh
bundle exec rake validate
```

This runs `tools/yaml/validate-header.rb` against all `.tex` pattern files and
reports any errors.

### 3. Regenerate derived files

```sh
bundle exec rake build
```

This runs all five generators:

| Generator | Produces |
|-----------|----------|
| `generate-converters.rb` | UTF-8 ↔ 8-bit encoding converters |
| `generate-pattern-loaders.rb` | Engine-aware `loadhyph-*.tex` files |
| `generate-ptex-patterns.rb` | pTeX-compatible patterns |
| `generate-tl-files.rb` | TeX Live `.tlpsrc` files and `language.dat` |
| `generate-plain-patterns.rb` | Plain-text `.pat.txt` / `.hyp.txt` files |

You can also run individual generators:

```sh
bundle exec rake converters
bundle exec rake loaders
bundle exec rake ptex
bundle exec rake texlive
bundle exec rake plain
```

### 4. Run tests

```sh
bundle exec rake spec          # RSpec unit tests
bundle exec rake               # validate + spec + build
```

### 5. Submit

Commit your changes and open a pull request. Each phase of changes should be a
separate commit.

## Updating patterns for an existing language

If you want to modify patterns for a language that already exists in the
repository, make sure your additions don't interfere with the current patterns.
It is **not** recommended to manually add patterns to a list generated by
`patgen`. Instead, modify the hyphenated word list that was used as a source and
regenerate the full pattern set.

This does not apply to hand-generated exception lists.

## Project layout reference

```
hyph-utf8/
├── source/generic/hyph-utf8/
│   ├── lib/tex/hyphen/         # Ruby library
│   ├── generate-*.rb           # Generator scripts
│   └── spec/                   # RSpec tests
├── tex/generic/hyph-utf8/
│   ├── conversions/            # Generated converters
│   ├── loadhyph/               # Generated loaders
│   └── patterns/
│       ├── tex/                # Source .tex pattern files (edit these)
│       ├── ptex/               # Generated pTeX patterns
│       ├── txt/                # Generated plain-text patterns
│       └── quote/              # Generated apostrophe quotes
TL/
├── tlpkg/                      # Generated .tlpsrc files
└── texmf-dist/                 # Generated language.dat
tools/
├── yaml/validate-header.rb     # YAML header validator
└── make_CTAN_zip.sh            # CTAN release packager
```

## Coding conventions

- Ruby code follows standard community style. No rubocop is configured yet.
- Generator scripts use explicit `TeX::Hyphen::` references (no top-level
  `include`).
- Tests use RSpec. Prefer `described_class` over hard-coded class references in
  specs.
- Pattern files use `%`-prefixed YAML comments for metadata headers.
